Procedure operative difensive

Playbook per incident response e CTI.

Scenari pratici con segnali, log da cercare, prime decisioni, evidenze da preservare e limiti. Nessuna procedura offensiva.

Ransomware

Segnali: file cifrati, note, accessi VPN anomali, discovery, compressione archivi, cancellazione backup.

Azioni: isolare host, preservare log, bloccare account sospetti, proteggere backup, attivare crisi e legale/privacy.

Log: EDR, AD, VPN, file server, proxy, DNS, backup, firewall, cloud.

Phishing credenziali

Segnali: email sospetta, link, login insoliti, MFA fatigue, regole mailbox e forwarding.

Azioni: revocare sessioni, reset password, controllare MFA, cercare destinatari simili, validare domini/URL.

Log: email gateway, IdP, M365/Google audit, proxy, DNS, EDR.

Account cloud

Segnali: impossible travel, app OAuth nuova, download massivi, token anomali, nuovi device.

Azioni: revoca token, rimozione app sospette, conditional access, audit mailbox/storage, reset credenziali.

Log: IAM, cloud audit, CASB, DLP, proxy, DNS.

Data exfiltration

Segnali: staging dati, archivi compressi, traffico anomalo verso cloud, domini rari, upload fuori baseline.

Azioni: blocchi mirati, preservazione evidenze, verifica dati coinvolti, DLP, comunicazione interna.

Log: proxy, DNS, firewall, DLP, cloud storage, file server.

CVE/KEV exploitation

Segnali: CVE in CISA KEV, asset esposto, log web anomali, processi o webshell sospetti.

Azioni: patch prioritaria, verifica compromissione, mitigazioni temporanee, hunting e WAF.

Log: web server, WAF, EDR, IAM, firewall, vulnerability inventory.

Business Email Compromise

Segnali: cambi IBAN, regole invisibili, accessi esteri, conversazioni alterate.

Azioni: congelare pagamenti, verificare out-of-band, audit mailbox, revoca sessioni, raccolta header.

Log: email, IdP, finance workflow, mailbox audit.

Schema comune di chiusura

Ogni playbook deve chiudersi con: evidenze preservate, timeline, IOC/IOA, tecniche MITRE, confidence, impatto, decisione, limiti e raccomandazioni.
